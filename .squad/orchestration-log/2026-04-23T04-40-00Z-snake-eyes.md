## Mandate

Eliminate `IServiceProvider` from the ViewModel layer by injecting specific dependencies into TopLevelCommandManager, TopLevelViewModel, and CommandProviderWrapper.

## Implementation

### Files Modified

1. **TopLevelCommandManager.cs**
   - Constructor now accepts 8 specific dependencies instead of `IServiceProvider`
   - Added `Lazy<HotkeyManager>` and `Lazy<AliasManager>` to break circular dependency
   - Resolved all service locator calls

2. **TopLevelViewModel.cs**
   - Constructor now accepts `HotkeyManager`, `AliasManager`, `ISettingsService`
   - Made `IContextMenuFactory` required (non-nullable)
   - Refactored `Hotkey` property to use injected managers with null-conditional operators
   - Refactored `AliasText` setter to use injected `AliasManager`

3. **CommandProviderWrapper.cs**
   - LoadTopLevelCommands and 4 other methods refactored to accept specific services
   - All explicit `GetService<T>()` calls replaced

4. **App.xaml.cs**
   - Updated DI registration with explicit factory lambda for TopLevelCommandManager
   - Registered `Lazy<HotkeyManager>` and `Lazy<AliasManager>` for dependency resolution

5. **Test Files** (3 files)
   - TopLevelCommandManagerTests.cs: Updated 12 tests
   - HotkeyManagerTests.cs: Updated 7 tests
   - DefaultContextMenuFactoryTests.cs: Updated 6 tests
   - BuildMockServices helper created to provide all required dependencies

## Key Technical Decisions

1. **Lazy<T> for circular dependency resolution:**
   - Both HotkeyManager and AliasManager depend on TopLevelCommandManager
   - TopLevelCommandManager needs both managers
   - Solution: Use `Lazy<HotkeyManager>` and `Lazy<AliasManager>` to defer resolution until first use
   - Verified zero runtime circular dependency issues

2. **Null-safe property access:**
   - Added null-conditional operators (`?.`) for HotkeyManager and AliasManager in TopLevelViewModel
   - Necessary because test mocks may pass null for optional dependencies

3. **Required IContextMenuFactory:**
   - Made non-nullable in TopLevelViewModel constructor to enforce proper DI
   - All callers must provide valid factory instance
