## Objective

Remove dead dependencies on TopLevelCommandManager from HotkeyManager and PinToCommand to unblock CommandRegistry extraction.

## Changes Implemented

### 1. HotkeyManager Dead Dependency Removal
- **File:** `src/modules/cmdpal/Microsoft.CmdPal.Common/HotkeyManager.cs`
- **Before:** Constructor accepted `TopLevelCommandManager _topLevelCommandManager` parameter; never used
- **After:** Parameter removed; field deleted
- **Impact:** Circular dependency broken (HotkeyManager → TLC)

### 2. PinToCommand Dead Dependency Removal
- **File:** `src/modules/cmdpal/Microsoft.CmdPal.UI/CommandPalette/CommandPaletteContextMenuFactory.cs` (PinToCommand inner class)
- **Before:** Inner class stored `TopLevelCommandManager` reference; never used in Pin/Unpin operations
- **After:** Field removed; all PinCommand/UnpinCommand methods operate directly on command operations
- **Impact:** Circular dependency broken (CommandPaletteContextMenuFactory inner → TLC)

### 3. TopLevelCommandManager Lazy<T> Changes
- **File:** `src/modules/cmdpal/Microsoft.CmdPal.UI.ViewModels/TopLevelCommandManager.cs`
- **Before:** Used `Lazy<HotkeyManager>` and `Lazy<AliasManager>` for circular dependency deferral
- **After:** Changed to direct injection (HotkeyManager and AliasManager properties injected directly)
- **Rationale:** Dead dependency removal eliminates the circular edges; lazy evaluation no longer needed
- **Benefit:** Cleaner dependency graph; direct access to manager instances

### 4. TopLevelCommandManager in DI Setup
- **File:** `src/modules/cmdpal/Microsoft.CmdPal.UI/App.xaml.cs`
- **Before:** Lazy<HotkeyManager> and Lazy<AliasManager> registered separately for TLC consumption
- **After:** HotkeyManager and AliasManager registered as singletons; TLC receives direct instances
- **Impact:** DI graph simplified; no lazy factory overhead
